### Secrets

<Alert type="info" className='my-2'>
    The **API token** option is recommended. API tokens are non-expiring and do not require storing your admin password in Homarr.
    Username and password authentication uses a session token that expires after inactivity, triggering a silent re-login on the next request.
</Alert>

<Alert type="warning" className='my-2'>
    The **DNS Hole Controls** widget (enabling, disabling, and timed disabling of blocking) requires the token to have settings-level permissions.
    The **DNS Hole Summary** widget (stats only) works with any valid token.
    If Homarr cannot read the blocking status due to insufficient permissions, the status indicator will be hidden but statistics will still be shown.
</Alert>
